# 智能出卷产品设计

> 2026-07-13 | 从 math-exam-generator 演化而来的产品构想

---

## 现在的工具 vs 智能产品

| | 现在的工具 | 智能产品 |
|---|---|---|
| 选题方式 | 标签匹配，随机抽 | 根据学生薄弱点，策略选取 |
| 难度 | 人工标1-5数字 | 根据正确率自动判断 |
| 去重 | 题目文字完全一样才去重 | 识别"换数字但本质相同"的题 |
| 学生感知 | 不知道谁在用 | 每个孩子独立画像 |
| 出卷策略 | 随机 | 薄弱优先 + 难度递进 + 易错重考 |
| 运行方式 | 单机HTML | 账号+云端，多设备同步 |

---

## 场景对比

### 现在的工具

```
你：我要10道乘法题
它：从"乘法"筐里随手抓10道，给你
你：再要5道小数题
它：从"小数"筐里随手抓5道，给你
```

它不知道孩子上周做过什么、哪里弱、题和题之间有没有重复。

### 智能产品

```
给小明出卷：

不出：
  ✗ 大量纯乘法题（他已经85分，熟了）
  ✗ 全是图形题（55分，太打击信心）

应该出：
  ✅ 小数加减多一点（62分，重点补）
  ✅ 图形题穿插2-3道（巩固但不过量）
  ✅ 乘法来两道保持手感
  ✅ 上次做错的"买东西找零"换场景重考

难度曲线：
  前几题简单 → 中间稍难 → 最后一道拔高
```

## 核心系统

### 1. 学生画像引擎

```
每个孩子的能力档案：
├── 各知识点掌握度（0-100%）
├── 易错类型标签
├── 做题速度曲线
└── 历史薄弱点变化趋势
```

来源：学生做过的每道题对错记录，自动积累。
不是人工填的。

### 2. 知识图谱

题和题之间的关系：

```
"A.买3斤苹果每斤5元花多少钱"  ← 最简单的乘法
    ↓ 加上"再买梨"
"B.买苹果又买梨，一共花多少"  ← 两组合并
    ↓ 加上"付钱找零"
"C.买两样东西付了20元找回多少" ← 两步变三步
    ↓ 加上"比谁贵"
"D.买两样东西，甲比乙多花8元，乙花了多少" ← 反向推理
```

工具知道：孩子不会做C，不是乘法不行，是"两步计算"没到位。
下次出题就多练两步计算，而不是给更多乘法题。

### 3. 出卷策略引擎

决定考什么、各占多少分的规则系统：

```
输入：学生画像 + 知识图谱 + 教学目标
 ↓
策略：哪些知识点要多考？哪些保持？哪些跳过？
 ↓
选题：从题库里挑——难度匹配、不重复、场景换新
 ↓
排布：先易后难、基础→核心→挑战
```

### 4. 批改系统

- 拍照/手动录入答案
- 自动判对错
- 错题自动归因到知识点
- 更新学生画像

```
┌─────────┐  ┌─────────┐  ┌──────────┐
│ 学生端   │  │ 家长端   │  │ 管理后台  │

└────┬────┘  └────┬────┘  └────┬─────┘
     └────────────┼─────────────┘
                  ↓
     ┌──────────────────────┐
     │       后端服务         │
     │  · 账号认证           │
     │  · 学生画像存储        │
     │  · 出卷策略计算        │
     │  · 做题记录分析        │
     └──────────┬───────────┘
                ↓
     ┌──────────────────────┐
     │       题库服务         │
```

## 从 MVP 起步的建议

只做**三年级数学、冀教版、一个功能**：

> 薄弱点优先出卷：孩子做了几套卷子后，系统知道哪里弱，下次自动多出那里的题。

验证这个功能真有用，再扩展。

### MVP 需要的

| 模块 | 做什么 | 技术选型 |
|---|---|---|
| 前端 | 学生做题界面 + 家长出卷界面 | 单页应用（Vue/React）或小程序 |
| 后端 | 账号、做题记录、出卷策略 | Node.js 或 Python |
| 数据库 | 学生画像、做题历史、题库 | PostgreSQL 或 Supabase |
| 部署 | 服务器或云服务 | Vercel/Railway 或自建 |

---

## 关键挑战

1. **题库建设**：需要大量带详细标签的题目（知识点、难度、易错类型），这是最大瓶颈
2. **知识图谱**：需要懂教材的老师参与，纯技术做不了
3. **冷启动**：新用户没有历史数据，拿什么出卷？（需要一份标准评估卷开局）
4. **算法验证**：自适应出卷是否真的比随机出卷效果好？需要对照实验

## 可参考的竞品

- 一起作业：侧重校内配套
- 猿题库：题库大但不对标具体教材
- 作业帮：搜题为主，出卷为辅
- 洋葱数学：视频课+练习，非出卷工具

机会点：**对标具体教材版本的智能出卷**，目前没有做得好的。
